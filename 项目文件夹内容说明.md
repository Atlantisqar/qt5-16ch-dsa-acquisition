# DSA 项目文件夹内容说明

说明：本文档根据当前仓库实际目录结构整理，重点介绍各文件夹的用途、包含内容，以及哪些目录属于源码、资源、第三方依赖或构建产物。

## 1. 项目整体概览

该项目是一个基于 Qt Widgets 和 CMake 的 16 通道动态信号采集与波形显示软件。当前实际参与构建的主源码位于 `src` 目录，图标和样式资源位于 `resources` 与 `icon` 目录，设备 SDK 位于 `third_party/device_sdk`，`build` 和 `build_x64` 为构建输出目录。

## 2. 顶层文件夹说明

### `.git/`

- 作用：Git 版本控制元数据目录。
- 包含内容：提交历史、分支引用、对象库、索引等内部管理文件。
- 备注：该目录由 Git 自动维护，不用于业务开发。

### `src/`

- 作用：当前主程序源码目录，也是 `CMakeLists.txt` 中明确加入编译的核心代码目录。
- 直接包含文件：
  - `main.cpp`：程序入口。
  - `mainwindow.h` / `mainwindow.cpp`：主窗口、菜单和页面调度。

#### `src/core/`

- 作用：采集核心逻辑。
- 代表内容：
  - `projectmanager`：项目文件创建、打开、保存。
  - `dsa16chtypes`：采集参数、DIO 配置等类型定义。
  - `dsa16chdllloader`：DLL 动态加载。
  - `dsa16chdeviceservice`：设备控制与 SDK 调用封装。
  - `circularbuffer`：环形缓冲区。
  - `multichanneldatastore`：多通道采集数据存储。
  - `acquisitionservice`：采集流程控制。
  - `plotservice`：绘图调度。

#### `src/dialogs/`

- 作用：对话框界面。
- 包含内容：
  - `aboutdialog`：软件信息对话框。
  - `newprojectdialog`：新建项目对话框。

#### `src/models/`

- 作用：数据模型。
- 包含内容：
  - `projectfilemodel`：项目文件、数据目录和配置项描述。

#### `src/pages/`

- 作用：主页面模块。
- 包含内容：
  - `homepage`：首页。
  - `settingspage`：参数设置页。
  - `acquisitionpage`：采集显示页。

#### `src/theme/`

- 作用：主题与视觉辅助。
- 包含内容：
  - `designtokens`：设计常量。
  - `themehelper`：样式辅助和阴影效果应用。

#### `src/widgets/`

- 作用：自定义控件。
- 包含内容：
  - `channelselectorwidget`：通道选择控件。
  - `menubutton`：侧边栏菜单按钮。
  - `waveformview`：单个波形绘制控件。
  - `waveformgridwidget`：多通道波形网格控件。

备注：`src` 是当前最应该关注和维护的源码目录。

### `resources/`

- 作用：Qt 资源文件目录，负责把样式、图标和 Windows 资源打包进程序。
- 包含内容：
  - `resources.qrc`：Qt 资源索引文件。
  - `app_icon.rc`：Windows 图标资源脚本。
  - `icons/`
    - `app_icon.ico`
    - `app_icon.png`
    - `logo.svg`
  - `styles/`
    - `main.qss`
- 备注：`resources.qrc` 中还通过别名引用了 `../icon` 下的一部分 PNG 图标。

### `icon/`

- 作用：原始 PNG 图标素材目录，供 Qt 资源系统引用。
- 包含内容：以菜单、功能按钮和品牌图片为主，例如：
  - `folder-add_line.png`
  - `folder-open_line.png`
  - `monitor_line.png`
  - `line-chart_line.png`
  - `setting_line.png`
  - `question-circle_line.png`
  - `cumtb2.png`
- 备注：主要是界面图标资源，不包含业务逻辑代码。

### `readme-image/`

- 作用：`README.md` 文档配图目录。
- 包含内容：
  - `项目管理.png`
  - `参数配置.png`
  - `实时采集与波形显示.png`
  - `软件说明.png`
  - `单通道采集绘图.jpg`
  - `四通道采集绘图.jpg`
  - `震动传感器校准仪测试.jpg`
- 备注：该目录服务于文档展示，不参与程序编译。

### `third_party/`

- 作用：第三方依赖目录。当前主要存放 16 通道动态信号采集板的设备 SDK。

#### `third_party/device_sdk/`

- 包含内容：
  - `atom_16ch_dsa_api.h`：设备接口头文件。
  - `atom_16ch_dsa_api.dll`：设备运行时动态库。
  - `16通道动态信号采集板接口函数说明.pdf`：接口说明文档。
- 备注：`CMakeLists.txt` 已将 `third_party/device_sdk` 加入头文件搜索路径，并在构建后优先从这里复制 DLL 到输出目录。

### `build/`

- 作用：构建输出目录，主要是 32 位或 Qt Creator 生成的中间文件与可执行产物。
- 包含内容：
  - `.cmake`
  - `CMakeFiles`
  - `Debug`
  - `Dsa16ChAcquisition.dir`
  - `Dsa16ChAcquisition_autogen`
  - `Desktop_Qt_5_15_2_MSVC2019_32bit-Debug`
  - `Desktop_Qt_5_15_2_MSVC2019_32bit-Release`
  - `win32`
- 备注：该目录属于自动生成内容，通常不建议手工修改；若清理后可重新通过 CMake 或 Qt Creator 生成。

### `build_x64/`

- 作用：64 位构建输出目录。
- 包含内容：
  - `CMakeFiles`
  - `Debug`
  - `Dsa16ChAcquisition.dir`
  - `Dsa16ChAcquisition_autogen`
  - `x64`
- 备注：与 `build` 类似，也属于自动生成的编译产物目录。

## 3. 根目录中与文件夹相关的重要补充

- `CMakeLists.txt`：项目构建入口，明确指定当前编译使用 `src`、`resources` 和 `third_party/device_sdk`。
- `README.md`：项目整体说明文档，配图来自 `readme-image` 目录。
- `read_dsa_data_and_plot.m`、`read_seismic_data_and_plot.m`：MATLAB 脚本，用于读取采集数据并绘图。
- `DSA_zh_CN.ts`：Qt 翻译源文件。
- 根目录下还保留了 `main.cpp`、`mainwindow.cpp`、`mainwindow.h`、`mainwindow.ui` 这一组旧版单窗口文件，属于历史遗留或早期界面实现；当前 `CMakeLists.txt` 未将这组文件纳入构建。
- 根目录下的 `atom_16ch_dsa_api.dll`、`atom_16ch_dsa_api.h`、`16通道动态信号采集板接口函数说明.pdf` 与 `third_party/device_sdk` 中对应文件内容一致，可视为兼容性保留或备份副本。

## 4. 结论

如果从开发维护角度看，重点目录是 `src`、`resources`、`icon` 和 `third_party/device_sdk`；如果从运行结果角度看，`build` 和 `build_x64` 只是构建输出目录；如果从文档展示角度看，`readme-image` 主要服务于 `README.md`。

后续若需要继续精简仓库结构，可以优先梳理根目录的旧版 `mainwindow` 相关文件和与 `third_party` 重复的 SDK 文件，但在确认兼容性之前不建议直接删除。
